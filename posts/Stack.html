Stack

In this chapter, we will work through a more or less realistic programming example on data structure. In computer science, it always come out with the abstract data type.
Sometimes choosing the best data structures or algorithms for a certain task is the key of success to developing high-performance software.

What's Stack?

A stack is an Abstract Data Type (ADT). Just like its name stack, it is just like a pile of coin kept on top of each other. Like a real-world stack, it only allows you to put a new coin on top of it, or remove the top coin.
In case you want to take out the coin at the bottom, you have to remove all the coins above of it. Generally, the last item that placed on top will be the first one to come out. This feature is known as Last-In-First-Out (LIFO). Some features (or simply said, some advantages)of stacks are listed below.

Features

We can declare a stack object or more specifically an abstract data structure(ADT) that allows us to do these operations:
  • Push(): Add element to top of the stack
  • Pop(): Remove element from top of stack
  • IsEmpty(): Check if the stack is empty, return boolean type result
  • IsFull(): Check if stack is full
  • Peek(): Get the value of the top element without remove it.

Implementation in Java

Here is the example of Generic Stack

public class GenericStack<E> {
    private java.util.ArrayList<E> list = new java.util.ArrayList<>();

    public int getSize() {
        return list.size();
    }
              
    public E peek() {
        return list.get(getSize() - 1);
    }
              
    public void push(E o) {
        list.add(o);
    }
              
    public E pop() {
        E o = list.get(getSize() - 1);
        list.remove(getSize() - 1);
        return o;
    }
              
    public boolean isEmpty() {
        return list.isEmpty();
    }
                
    @Override
    public String toString() {
        return "stack: " + list.toString();
    }
}

What is a Stack Overflow Error?

A stack overflow occurs if the call stack pointer exceeds the stack bound.

Bad recursive call will cause this error happened, if your recursive functions don't have the correct termination condition, it ends up for calling itself forever.
